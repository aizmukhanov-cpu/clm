# prisma/scripts

Утилитные TS-скрипты, выходящие за рамки `prisma/seed.ts`. Запуск через `dotenvx run -f .env.local -- tsx prisma/scripts/<file>` (или `tsx` напрямую, если переменные уже в окружении).

## Активные

- **`update-passwords.ts`** — сбрасывает пароли всех пользователей на `password123`. Только для dev-окружения.

## archive/

Скрипты, уже выполненные на текущей БД и оставленные как историческая запись. Перезапускать без необходимости не нужно.

- `add-users.ts`, `add-users-pg.ts` — добавили B2B/KM/KAM пользователей и переназначили клиентов конкретным менеджерам. Hardcoded ID/ИНН.
- `rebalance-kam.ts` — однократное перераспределение портфеля `user-kam` между другими KAM.
- `migrate-manual.ts` — **внимание:** содержит единственный источник истины для двух изменений схемы, которых нет в `prisma/migrations/`:
  - добавление `ESCALATED` в enum `TaskStatus`
  - создание таблицы `proposal_templates`

  Перед накатом схемы на новое окружение эти изменения надо либо перенести в нормальную Prisma-миграцию, либо выполнить `migrate-manual.ts` руками.
